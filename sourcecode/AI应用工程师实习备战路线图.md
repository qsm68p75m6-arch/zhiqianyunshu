# AI 应用工程师（实习生）岗位备战路线图

## 总体策略：2个深度项目 + 1个微调项目 + 工程化包装

不用追求数量，**3个项目打透**比10个Demo强百倍。

---

## 第一阶段（第1-5周）：夯实基础 + 启动核心项目

### 周目标

| 周 | 重点 | 输出 |
|---|------|------|
| 1-2 | Python工程化（FastAPI/Git/项目结构）+ 大模型基础概念 | 能独立搭建一个规范的Python项目 |
| 3-4 | 深入学习LangChain/LlamaIndex + 向量数据库 | 跑通一个最小RAG demo |
| 5 | **启动Project 1：RAG知识库系统** | 完成技术选型和数据准备 |

### 关键动作

- 用FastAPI封装一个大模型调用接口，练习错误处理/日志/配置管理
- 理解Transformer自注意力、位置编码、KV-Cache（面试必问）
- 选定Project 1的领域方向（建议：**你熟悉的某个垂直领域**，如课程资料/某产品文档/法律条文）

---

## 第二阶段（第6-10周）：Project 1 深度打磨（RAG知识库）

这是**最对口JD的核心项目**，必须做到"能讲清楚每个决策"。

### 需要覆盖的技术点

```
文档解析 → 文本切分(对比多种策略) → Embedding选型 → 向量数据库(推荐Chroma/Milvus) 
→ 检索策略(BM25+向量混合) → 重排序 → Prompt优化 → 生成
```

### 必须有的评估闭环

- 构建 **200-500条** 人工标注的问答对作为评估集
- 记录指标：准确率、召回率、幻觉率
- **至少做3轮迭代**，记录每次改动和效果变化

### 简历写法示例

> \"基于XX领域500份文档搭建RAG知识库系统，通过递归字符分割+语义分块混合策略，结合BM25+MTEB向量混合检索与bge-reranker重排序，将Top1回答准确率从58%提升至89%（基于320条测试集）。系统支持多轮对话与来源追溯，使用FastAPI部署，平均响应时间<2s。\"

## 第三阶段（第11-14周）：Project 2 - Agent/工作流项目

### 选项A（推荐）：业务自动化Agent

- 场景举例：数据分析助手 / 日报自动生成 / 简单审批流程机器人
- 技术栈：LangGraph 或 Dify
- 核心能力体现：工具调用设计、多步规划、错误恢复

### 选项B：如果你有特定兴趣方向

- NL2SQL Agent（对接真实数据库）
- 代码审查Bot（接入Git仓库）
- 内部FAQ智能客服

### 关键要求

- **必须有真实的"业务价值"描述**（节省了多少时间/替代了什么重复劳动）
- 展示工具调用的完整链路和异常处理

---

## 第四阶段（第15-18周）：Project 3 - LoRA 微调

### 推荐方案

```
基座模型：Qwen2.5-7B-Instruct（或DeepSeek-R1-Distill-Qwen-7B）
任务：指令微调（如特定风格对话/领域问答/代码生成）
方法：QLoRA (4bit量化 + LoRA rank=16)
框架：unsloth + transformers
数据：自己构建 1000-3000 条高质量指令数据
```

### 必须记录的内容

- 数据构建过程（来源、清洗规则、质量把控）
- 超参数实验（lr、batch_size、rank 的不同组合及效果）
- **前后对比**（用同一套评估集，量化提升幅度）

### 面试杀手锏话术

> \"在Qwen2.5-7B基础上用QLoRA进行指令微调，rank=16, lr=2e-4, 使用1500条自建领域数据训练3个epoch。通过消融实验发现lr=2e-4时收敛最快且无过拟合，人工评分从3.2/5提升至4.3/5，领域问题准确率提升37%。\"

### GitHub仓库规范

```
project-name/
├── README.md          # 项目介绍 + 效果数据 + 架构图
├── docs/              # 设计文档、评估报告
├── src/               # 模块化源码
├── configs/           # 配置文件
├── data/              # 示例数据（不含敏感信息）
├── evaluations/       # 评估脚本和结果

```

## 三个核心建议

**1. 不要同时开工多个项目** —— 一个做深再做下一个，面试时能讲20分钟的项目胜过只能讲5分钟的5个项目。

**2. 每个项目都要有"为什么"的故事** —— 为什么选这个Embedding？为什么用这种切分策略？为什么lr设为2e-4？这些决策过程才是面试官想听的。

**3. 数据和评估是你的秘密武器** —— 绝大多数候选人的项目没有评估闭环。你有，你就已经超过了80%的人。
